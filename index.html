This simulator shows the exact flow you described: a knowledge base is chunked, converted into simple vector-like representations, the user query is embedded too, top matching chunks are retrieved, and the final prompt is augmented before the model generates an answer.

2) RAG Workflow

The highlighted step shows what the pipeline is currently doing.

1Chunk the knowledge base

Split long content into smaller chunks so retrieval is more precise and the prompt stays manageable.

2Create embeddings for chunks

In real RAG, each chunk becomes a numeric vector. Here, we simulate that with important terms.

3Embed the user query

The query is represented in the same space so we can compare it with stored chunk vectors.

4Retrieve top matching chunks

The most similar chunks are selected from the vector store.

5Augment the prompt

Retrieved context is appended with the user query before generation.

6Generate grounded answer

The model answers using only the retrieved information plus the question.

Blue highlight = current step being processed
Green = step completed
Long context can increase cost and reduce focus, so RAG retrieves only relevant parts
